핵심 메시지

이번 수업의 목표는 AI 도구 사용법을 더 많이 아는 것이 아니라, 신세계 I&C가 현업의 업무 문제를 좁고 깊게 정의하고 조직 안에 남는 AX 산출물로 전환하는 감각을 만드는 것입니다.
Prompt

업무 지시서

AI에게 시킬 일을 목적, 입력 자료, 산출물, 검토 기준으로 구조화합니다.

Context

작업 책상

자료, 용어, 정책, 예시, 금지 사항을 반복 가능한 Context Pack으로 묶습니다.

Harness

운영 장치

개인 사용을 권한, 로그, 품질 평가, 승인 지점이 있는 업무 루프로 연결합니다.

Loop

학습 구조

AI 초안과 사람의 수정 결과를 비교해 다음 실행 기준을 개선합니다.

오후 수업 흐름

기술 소개에서 끝내지 않고 업무 전환 질문으로 이어갑니다.

Part 1

AI 기술의 속도와 방향

생성형 AI, 멀티모달, Agent AI, Physical AI를 조직 운영 변수로 읽습니다.

Part 2

AI Native & AI Harness 사례

MAX Studio, MAX Tutor, MAX Cast 사례를 통해 교육, 상담, 콘텐츠, 운영 자동화 구조를 봅니다.

Part 3

우리 업무에 어떻게 도입할까

문제 찾기, 빠른 구현, 검증 체계, PoC, PM 전환으로 이어지는 AX 전환 절차를 정리합니다.

Part 4

AX 도입 전략과 리스크 관리

내부 전문가 양성, 좁고 명확한 과제 정의, 사람 검토와 보안 원칙을 실행 기준으로 만듭니다.

복습 지도

강의 후에는 여섯 개 질문으로 다시 보면 됩니다.

01

AI는 얼마나 빨리 변하고 있나?

지식 비용, 멀티모달, Agent AI 흐름을 업무 설계의 전제로 둡니다.

02

도구 사용과 AI 직원/팀은 무엇이 다른가?

개인 생산성보다 업무 루프, 품질 기준, 책임 구조가 있는지 봅니다.

03

신세계 I&C 업무에는 어디부터 붙일까?

반복 빈도, 자료 준비도, 시스템 연결성, 보안 리스크로 후보를 좁힙니다.

04

Prompt, Context, Harness, Loop를 어떻게 나눌까?

요청서, 작업 책상, 운영 장치, 개선 리듬을 분리해 설계합니다.

05

AX는 어떤 순서로 도입해야 하나?

교육, 업무 발굴, 도메인 전문가 양성, 90일 PoC, 추진안으로 이어갑니다.

06

리스크는 언제 다뤄야 하나?

데이터, 권한, 품질, 책임, 감사 기준을 PoC 설계 초기에 고정합니다.

서울대학교 EPM 연결점

EPM 수업은 네트워크 안의 실행 과제를 발견하는 자리입니다.

공개 강의 문안은 기술 교육처럼 보이더라도, 현장에서 남겨야 할 결과는 AI4PM 실행 과제 후보입니다. 신세계 I&C의 역할은 그룹과 고객사의 현업 문제를 기술 언어로 번역하고, 내부 전문가가 검증 가능한 PoC 범위로 좁히는 것입니다.

수업 공통 언어 형성
진단 업무 후보 분류
과제 PoC 후보 3개 선정
전환 90일 실행안 작성

신세계 I&C 적용 렌즈

업무 영역을 넓게 말하지 말고, 검증 가능한 장면으로 줄입니다.

고객 접점

  • 상담, FAQ, VOC, 멤버십 문의처럼 반복되는 질문과 응답을 구조화합니다.
  • AI 답변은 초안으로 두고, 정책/약관/민원성 판단은 사람이 승인합니다.

매장/물류 운영

  • 현장 이슈, 운영 매뉴얼, 재고/배송 예외 처리처럼 지식 검색 시간이 긴 업무를 찾습니다.
  • PoC는 샘플 데이터와 공개/비식별 자료로 먼저 검증합니다.

IT서비스 운영

  • 장애 리포트, 티켓 분류, 변경 영향 검토, 사내 지식 검색을 Context Pack으로 묶습니다.
  • 로그, 권한, 검토자, 에스컬레이션 기준을 Harness에 포함합니다.

경영/기획 보고

  • 시장 자료, 프로젝트 현황, 회의록, KPI를 보고 초안과 질문 리스트로 바꿉니다.
  • 수치와 의사결정 문장은 원자료 대조와 책임자 검토를 통과해야 합니다.

현장 실습

내 업무 하나를 AI Native 업무 단위로 바꿉니다.

Lab 1

업무 병목 캔버스

반복 빈도, 처리 시간, 오류 비용, 고객/현업 영향도를 기준으로 후보 업무를 고릅니다.

  • 문제 문장
  • 현재 프로세스
  • 사람 검토 지점
Lab 2

Prompt와 Context Pack

업무 요청서와 자료 책상을 분리해 AI가 같은 기준으로 반복 실행할 수 있게 만듭니다.

  • 입력 자료
  • 출력물 형식
  • 금지 사항
Lab 3

Harness와 검증 기준

개인 실험을 조직 업무로 쓰기 위해 로그, 권한, 평가 기준, 승인자를 정합니다.

  • 품질 기준
  • 검토자
  • 에스컬레이션
Lab 4

90일 PoC 후보 선택

ROI와 실행 난이도를 함께 보고, 작게 만들 수 있는 후보 1-3개로 줄입니다.

  • 검증 범위
  • KPI
  • 내부 오너

90일 실행 프로그램

교육 이후 남겨야 할 것은 학습 완료가 아니라 작동 산출물입니다.

1-2주

공통 언어 형성

AI 리터러시, AI Native, Harness, 보안 원칙을 같은 언어로 맞춥니다.

3-4주

적용 업무 발굴

업무 후보를 모으고 ROI와 실행 난이도로 PoC 후보 3개를 우선순위화합니다.

5-8주

도메인 전문가 설계

Context Pack, 업무 루프, 품질 평가표, 사람 승인 기준을 만듭니다.

9-12주

제작과 검증

시제품, 테스트 케이스, KPI 리포트, 본 프로젝트 전환안을 남깁니다.

Guardrails

실습 중 지킬 보안 원칙

  • 실제 고객 개인정보, 결제 정보, 계정 정보, 운영 로그 원본은 사용하지 않습니다.
  • 사내 소스코드, 접근키, 비공개 시스템 문서, 계약/재무 민감 자료는 외부 도구에 넣지 않습니다.
  • AI 결과는 정책 판단, 보안 판단, 대외 보고의 최종본이 아닙니다.
  • 사람의 검토와 승인 지점을 먼저 정한 뒤 업무 루프에 AI를 넣습니다.
